PHI Data Handling Policy

1. Purpose

This policy defines the requirements for handling Protected Health Information (PHI) within PHC Nexus and all connected systems. It applies to all employees, contractors, and system users who may access, process, or store PHI data.

2. Scope

This policy covers:

  • All digital systems processing patient or health-related data
  • Internal tools and platforms (including PHC Nexus)
  • Data exports, backups, and archival processes
  • Third-party integrations that may access PHI

3. Classification Rules

All data entities must be classified using one of the following levels:

Classification Description Access Level Export
PHI Contains protected health information Restricted + Audited Blocked in standard export
Unknown Not yet classified Treated as PHI Blocked in standard export
Non-PHI No health information Standard role-based Allowed

4. Access Controls

  • PHI data access requires explicit authorization beyond standard role membership
  • All PHI read and write operations must generate audit trail entries
  • Guest users are permanently blocked from PHI data access
  • Download and inline preview of PHI attachments are treated as read access events

5. Audit Requirements

  • All access to PHI entities must be logged (create, read, update, delete)
  • Audit logs are immutable and retained for 6 years
  • Audit logs must not duplicate PHI content — only reference entity IDs
  • pgAudit provides secondary DB-level verification

6. Export & Download Rules

  • Standard exports exclude all PHI and Unknown classified data
  • GDPR self-service export follows separate compliance flow
  • Regulated/admin export of PHI requires explicit approval and full audit trail
  • Bulk attachment export is blocked for PHI content in standard flows

7. Incident Response

In case of suspected PHI data breach:

  1. Immediately notify the Data Protection Officer
  2. Preserve all audit logs related to the incident
  3. Document the scope and timeline of potential exposure
  4. Follow the Incident Response Runbook (link)

8. Review & Updates

This policy is reviewed quarterly. All changes require approval from the Data Protection Officer and Head of Compliance.
