Pozor: Zmena workflow priradi existujici ukoly do vychoziho stavu (Otevreny). Ukoly ve stavech, ktere v nove sablone neexistuji, budou automaticky prerazeny. Tuto akci nelze vzit zpet.

Rezim pouziti

Pouzit sablonu
Projekt sleduje sablonu. Zmeny v sablone se automaticky propagují.
Vlastni workflow (fork)
Vytvori kopii sablony. Dalsi zmeny sablony se nepromitnou.

Migracni strategie

▼
// Migrace: existujici per-project workflows zustávaji jako "custom" (fork). // Systemove seedy (Software, TaskManagement, Approval, Custom) // se stanou workflow_templates s is_system = true. Schema::create('workflow_templates', function (Blueprint $table) { $table->uuid('id')->primary(); $table->string('name'); $table->string('slug')->unique(); $table->text('description')->nullable(); $table->boolean('is_system')->default(false); $table->string('category', 50); $table->foreignUuid('author_id')->nullable()->constrained('users'); $table->string('version', 20)->default('1.0'); $table->timestamp('published_at')->nullable(); $table->timestamps(); $table->softDeletes(); }); // Pridat FK do projects tabulky Schema::table('projects', function (Blueprint $table) { $table->foreignUuid('workflow_template_id') ->nullable() ->constrained('workflow_templates') ->nullOnDelete(); });
