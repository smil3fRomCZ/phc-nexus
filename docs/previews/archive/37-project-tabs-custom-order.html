Data model

Pořadí se ukládá jako JSONB sloupec tab_config na tabulce projects. Schéma: 

projects.tab_config (jsonb, nullable)
{
  "order": ["overview", "board", "table", "epics", "approvals", "members",
            "time", "reports", "history", "workflow"],
  "hidden": ["estimation", "gantt", "wiki"]
}

// Pokud sloupec == NULL → default pořadí (jako dnes)
// "overview" je vždy na první pozici, backend ho vynucuje
// Nové taby přidané do systému (např. Goals v budoucnu) se automaticky
// zařadí na konec viditelných (před overflow čáru)

Migration

Schema::table('projects', function (Blueprint $table) {
    $table->jsonb('tab_config')->nullable();
});

Backend API

PUT  /projects/{project}/tab-config    → { order: [...], hidden: [...] }
                                         → Gate: manageProject
                                         → Audit entry: tab_config_updated
DELETE /projects/{project}/tab-config  → reset na default

Frontend

• tabConfig přidáno k project shared props
• ProjectTabs.tsx nejdřív aplikuje custom pořadí + odfiltruje hidden, pak měří overflow
• ProjectTabsCustomizeModal.tsx s drag & drop (@dnd-kit/sortable)
• "Obnovit výchozí" posílá DELETE
• Pořadí je per-projekt, nikoliv per-user — změna ovlivní všechny členy projektu
