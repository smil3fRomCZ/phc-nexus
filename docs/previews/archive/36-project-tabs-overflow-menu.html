Bod 36 — Project tabs overflow menu + chip v project headeru

Při 13 tabech + chipu "Poslední update" se obsah na běžném laptopu (1280–1440px) nevejde a pravá strana se ořízne. Návrh: (1) přesunout chip z tab baru do project headeru (vedle ESHOP / Aktivní badge), (2) implementovat container-aware overflow menu — taby, které se nevejdou, spadnou do dropdown "Další ▾", (3) na mobilu (<640px) ikona-only s fade indikátorem pro scroll.

Implementace — co je potřeba

Komponenta Změna Složitost
ProjectHeaderCompact.tsx Přidat lastUpdate prop + chip do layoutu (header-spacer + chip) ~20 řádků
ProjectTabs.tsx Odstranit chip (přesunut do headeru) -15 řádků
ProjectTabs.tsx ResizeObserver + měření šířky položek, overflow detection, dropdown state ~80 řádků
ProjectTabs.tsx Fade gradient pro mobile (jen CSS) ~5 řádků
Backend Nic — projectLastUpdate už je shared prop 0

💡 Alternativa (jednodušší, ~20 řádků): jen přesun chipu do headeru + fade gradient na všech breakpointech. Bez overflow menu, spolehne se na horizontální scroll. Funguje, ale uživatel musí scrollovat a nemá hint, které taby jsou "skryté".

💡 Doporučuji plný návrh (overflow menu) — jednorázová investice ~100 řádků, ale UX je výrazně lepší a škáluje to i kdyby přibyly další taby v budoucnu.
